Comand injection attack

  1. 정의 : 웹을 통해 시스템 명령어를 실행하는 공격 웹 내부에서 시스템 명령어를 실행하는 경우, 입력 값을 제대로 검사하지 않으면, 해커 마음대로 시스템 명령어를 실행한다.
  2. 리눅스는 ‘;’를 이용하여(윈도우에서는 &&이다.) 명령어를 동시에 여러 개 실행할 수 있다. 그래서 A;B형식으로 A에 서버가 원하는 명령어를 넣고 B에 해커가 원하는 명령어를 넣어서 공격할 수 있다.
  3. 리눅스에서 '&&'를 사용해서 명렁어를 동시에 여러 개 실행할 수 있다. '&&'는 ';'와 달리 앞에 명령어가 실패하면 뒤에 명령어가 실행되지 않는다.
  4. ‘|’(파이프)를 이용하여 공격할 수도 있다. 파이프는 앞 명령어의 출력 결과 값을 뒤 명령어 입력 값에 넣어주는 명령어다.
  5. ‘&’를 이용해서 공격할 수 있다. ‘&’는 앞에 명령어를 백그라운드에서 실행하게 하는 명령어다. 앞에 명령어는 백그라운드에서 실행하게 하고 뒤에 명령어는 앞에서 실행하게 한다.
  6. 대응 : 입력 값을 제대로 검증해야 하고, 불필요한 문자들은 X => ''로 없애야 한다.
