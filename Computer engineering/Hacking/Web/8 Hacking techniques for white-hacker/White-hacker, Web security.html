White-hacker, Web security

  1. 보안 취약점과 해킹
  2. 웹 보안의 중요성
    • 웹은 방화벽에 대해서 항상 열려 있다.
    • 모의 해킹의 관문이다.
    • 모바일 앱도 웹과 통신한다.
  3. 웹 아키텍쳐
    • 로직 티어는 웹사이트의 각종 메뉴들을 서비스하고 클라이언트의 요청을 처리해준다.
    • 데이터 티어는 로직 티어의 요청에 따라서 데이터를 돌려주고 로직 티어는 이 데이터를 가공하여 최종적으로 클라이언트에게 전달한다.
  4. 세션 유지와 쿠키
    • HTTP는 stateless이기 때문에 세션 ID를 사용해서 이전 상태를 유지한다.
    • 세션 ID는 보통 쿠키로 전달한다.
    • 쿠키란?
      • 변수 = 값의 집합
      • 서버는 Set-Cookie 헤더를 이용해 쿠키를 전달하고, 브라우저에서 이를 저장해두었다가 Cookie 헤더에 전송한다.
    • 세션 ID를 탈취함으로써 세션 하이재킹 공격이 가능한다.
  5. Http 프록시를 이용하면 Http 요청과 응답을 중간에서 가로챌 수 있다. 메시지 내용을 볼 수 있고 수정해서 보낼 수가 있다. 웹 해킹이나 웹 보안 취약점을 찾을 때 아주 유용하다. (대표 프로그램 Burp Suite)
  6. 웹 서버를 다시 실행시키려면 # /opt/lampp/lampp restart를 입력해야 한다.
